## Project Overview

**Fixit 24/7** is a full-stack on-demand home repair marketplace connecting homeowners with tradies (tradespeople). It is a pnpm monorepo using TypeScript project references.

## Monorepo Structure

```
artifacts/
  api-server/     Express 5 REST API (Node.js)
  fixit247/       React 19 frontend (Vite)
  mockup-sandbox/ Design system sandbox
lib/
  db/             Drizzle ORM schema + PostgreSQL (source of truth)
  api-spec/       OpenAPI 3.1 spec (single source of truth for API contracts)
  api-client-react/ Auto-generated React Query hooks (Orval → do not edit manually)
  api-zod/        Auto-generated Zod validators (Orval → do not edit manually)
scripts/          Seed data and utilities
```

## Commands

### Root (run from repo root)
```bash
pnpm build             # Build all packages
pnpm run typecheck     # Type-check entire workspace
```

### API Server
```bash
pnpm --filter @workspace/api-server run dev        # Dev server
pnpm --filter @workspace/api-server run build      # Build (esbuild → dist/index.mjs)
pnpm --filter @workspace/api-server run typecheck
```

### Frontend
```bash
pnpm --filter @workspace/fixit247 run dev          # Vite dev server (port 5173)
pnpm --filter @workspace/fixit247 run build        # Build → artifacts/fixit247/dist/public
pnpm --filter @workspace/fixit247 run typecheck
```

### Database
```bash
pnpm --filter @workspace/db run push               # Apply schema changes
pnpm --filter @workspace/db run push-force         # Force push
```

### Code Generation (after editing openapi.yaml)
```bash
pnpm --filter @workspace/api-spec run codegen      # Regenerate api-client-react + api-zod
```

### Seeding
```bash
pnpm --filter @workspace/scripts run seed-demo     # Load realistic demo data (idempotent)
pnpm --filter @workspace/scripts run seed-products # Seed service categories
```

## Architecture

### API Contract Workflow
The OpenAPI spec at `lib/api-spec/openapi.yaml` is the **single source of truth**. Do not edit files in `lib/api-client-react/src/generated/` or `lib/api-zod/src/generated/` — they are regenerated by Orval from the spec. The workflow is:
1. Edit `openapi.yaml`
2. Run `codegen` to regenerate clients
3. Implement the route in `artifacts/api-server/src/routes/`

### API Server (`artifacts/api-server`)
- Express 5 app with all routes prefixed `/api/`
- Stripe webhook middleware runs **before** `express.json()` to preserve raw body for signature verification (`webhookHandlers.ts`)
- Rate limiting: 10 req/min on auth endpoints, 30 req/min on write endpoints
- Structured JSON logging via pino-http
- WebSocket server at `/api/ws?token=JWT` managed by `src/lib/ws-manager.ts`
- Job matching engine in `src/lib/matching.ts` — sends fire-and-forget match emails via SendGrid

### Frontend (`artifacts/fixit247`)
- Routing: Wouter (not React Router)
- Auth state: Zustand store via `useAuth` hook; token getter injected via `setAuthTokenGetter()` in `main.tsx`
- Data fetching: generated React Query hooks from `@workspace/api-client-react`
- UI: Tailwind CSS 4 + shadcn/ui + Framer Motion

### Database (`lib/db`)
Key tables and relationships:
- `users` — single `role` enum (`homeowner` | `tradie` | `admin`); tradie-specific fields inline
- `jobs` → `claims` → accepted claim triggers auto-creation of `conversations`
- `messages` — polled every 5s (no persistent WebSocket store)
- `walletBalances` + `walletTransactions` — tradies get $111 AUD (`WELCOME_GRANT_CENTS = 11100`) on signup, spend $22 default per claim (`LEAD_COST_CENTS_DEFAULT = 2200`); welcome grant renews monthly for 6 months at midnight AEST on the 1st (node-cron); all amounts stored in cents
- Emergency membership fields on `users` (e.g., `emergency_membership_active`, `emergency_calls_used_this_year`)

### Stripe Integration
Handles two flows:
1. **Tradie credits**: package purchases ($49/$99/$149 AUD)
2. **Emergency membership**: homeowner subscription ($49 AUD/month); detected by subscription ID lookup in `stripeStorage.ts`

Uses `stripe-replit-sync` for webhook handling in addition to standard Stripe webhooks.

### Service Area Matching
Tradies set `serviceSuburbs` (tag list) + `serviceRadius` (km). The matching engine filters new jobs against tradie service areas and sends match emails.

## Environment Variables

Required:
- `SESSION_SECRET` — JWT signing key
- `DATABASE_URL` — PostgreSQL connection string
- `PORT`

Optional:
- `STRIPE_SECRET_KEY`, `STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET`
- `SENDGRID_API_KEY`, `FROM_EMAIL`, `SENDGRID_*_TEMPLATE_ID`
- `OPENAI_API_KEY` — for AI-assisted job descriptions
- `ALLOWED_ORIGINS` — comma-separated CORS origins
